Configuration Paperless-ngx
Liaison active Non configuré

Configurez et testez la liaison avec le système d'archivage documentaire intelligent Paperless-ngx. Une fois configurée, les documents signés peuvent être archivés dans Paperless et les fichiers locaux supprimés.

URL de base de votre instance Paperless-ngx, sans slash à la fin.
Générez un token dans Paperless : Profil → Jeton d'API.
Champs personnalisés Paperless (traçabilité)

Créez ces champs dans Paperless (Administration → Champs personnalisés) puis renseignez ici leurs identifiants numériques. Laissez vide les champs non utilisés : ils seront simplement ignorés lors de l'archivage.

Champ integer — ID Paperless du document original (avant signature). Positionné sur le document signé.
Champ integer — ID Paperless du document signé. Positionné en retour sur le document source.
Champ integer — ID de la demande de signature dans eSign.
Champ date — Date à laquelle le document a été signé.
Champ string — Email(s) des signataires, séparés par des virgules.
API d'archivage vers Paperless

Une fois la demande de signature terminée, votre application métier appelle l'une des API suivantes :

Archivage automatique (recommandé)

POST /ws/signrequests/{id}/archive-to-paperless
Header : x-api-key: votre-token-api-esup

eSign récupère le PDF signé, le transfère dans Paperless avec les métadonnées du document source, puis supprime les fichiers locaux. Retourne { paperlessDocumentId, paperlessUrl, signRequestId }.

Liaison manuelle (si l'upload Paperless est géré côté métier)

POST /ws/signrequests/{id}/clean-and-link?externalUrl={url-paperless}
Header : x-api-key: votre-token-api-esup
  • {id} : identifiant de la demande de signature (disponible dans le callback REST)
  • {url-paperless} : URL de téléchargement dans Paperless (ex: https://paperless.exemple.fr/api/documents/42/download/)
  • Le token x-api-key est géré dans Tokens API
